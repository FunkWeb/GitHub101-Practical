## A
- **API:** - a set of rules to let different programms to exchange data and communicate

## B
- **Branch** - create an independent branch so you can make changes without affecting the main project

## C
- **Commit** - saves changes to files

## F
- **Fork** - create a copy of a repository
- **Fetch** - gets updates/changes from a repository

## M
- **Merge** - integrates changes from one branch into another. 
- **Merge Conflict** - 2 different branches make different changes to the same file

## P
- **Pull Request** - request to add changes to a branch/project
