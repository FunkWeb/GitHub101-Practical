# Retningslinjer for bidrag

## Branch-navn
Bruk formatet: `feature/<ditt-navn>-<kort beksrivelse>`

## Commit-stil (enkel)
Start meldingen med type:
- `feat:` for ny funksjon
- `fix:` for retting
- `docs:` for dokumentasjon
- `chore:` for små endringer

## Sjekkliste for Pull Request
- [ ] Jeg brukte en egen branch
- [ ] Jeg hentet oppdateringer fra 'upstream' før push
- [ ] Jeg løste eventuelle konflikter
- [ ] CI-testene er grønne (ingen 'TBD' igjen)

## Konflikt-policy
Instruktøren endrer med vilje en linje i dokument/kode underveis.
Husk å hente fra `upstream` og rebase/merge **før** siste push.
